## 🎯 RESEARCH OBJECTIVE
Solve specific identified problems with focused research and validation. Prioritize concrete solutions over exploratory research.

## 📋 EVOLVE METHODOLOGY

### Phase 1: Codebase Investigation
- Analyze provided codebases to understand architecture, patterns, and data flows
- Document key modules, interfaces, and dependencies
- Identify specific problems and their root causes
- Generate focused documentation for the problem domain

### Phase 2: Research Planning
Apply targeted reasoning patterns to generate focused research queries:
- **Problem Analysis**: What is the root cause of the specific issue?
- **Known Solutions**: What existing solutions have been tried for similar problems?
- **Constraint Mapping**: What constraints must the solution satisfy?
- **Risk Assessment**: What are the potential failure modes?
- **Validation Planning**: How will we verify the solution works?

### Phase 3: Deep Research
- Execute focused research on the specific problem domain
- Survey solutions that have worked for similar problems
- Synthesize findings into actionable solution approaches
- Focus on proven techniques with clear applicability

### Phase 4: Proposal Generation
- Generate concrete solution proposals from research findings
- Include implementation details and risk mitigation
- Specify expected outcomes and success criteria
- Target direct problem resolution

### Phase 5: Merge & Synthesize
- Select the most promising solution approach
- Resolve implementation details and edge cases
- Create implementation plan with clear milestones

### Phase 6: Implementation
- Implement the proposed solution with clean interfaces
- Follow existing codebase conventions
- Include documentation and error handling

### Phase 7: Experiments
- Validate the solution against the original problem
- Measure relevant success metrics
- Verify no regression in other areas

### Phase 8: Analysis
- Confirm the problem is solved
- Document any remaining edge cases
- Identify any secondary issues introduced
- **Determine if follow-up work is needed**

## 🔄 ITERATION POLICY
After analysis, if the problem is not fully resolved:
- Focus next iteration on remaining gaps
- Apply targeted fixes to specific issues
- Validate complete problem resolution

## ✅ SUCCESS CRITERIA
- Original problem is demonstrably solved
- Solution is well-documented and maintainable
- No significant regressions introduced
- Edge cases are handled appropriately
- Solution is validated with concrete evidence
